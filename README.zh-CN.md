# Pedit 中文说明

Pedit 是一个运行在 Codex 里的本地图片编辑插件。它把“图片画布、局部标注、版本管理、参考图、修图任务交接”整合到一个工作流里，让你可以用更自然的方式和 Codex 一起修图。

简单说：你把图片放进 Pedit，在画布里写清楚想改哪里、怎么改；Pedit 会把图片、选区、参考图和指令整理成任务交给 Codex；Codex 完成后，结果会回到 Pedit 的版本树里，方便继续修改、对比和导出。

## 适合谁用

- 想用 AI 修图，但希望保留原图版本和修改历史的人
- 经常需要局部改图、换文字、调色、换背景、合成参考图的人
- 希望在 Codex 里完成图片编辑工作流，而不是反复下载、上传、复制文件的人
- 想把一次修图拆成多个版本，方便比较、回退和继续分支探索的人

## 主要功能

- 图片项目管理：可以创建、进入和管理不同图片项目
- 无限画布：在画布里查看当前图片和历史版本
- 版本树：每次生成结果会成为一个版本节点，方便回退和分支
- 局部标注：可以圈选区域，并写局部修图指令
- 整图编辑：不圈选区域时，按整图优化流程处理
- 参考图上传：可以上传参考图，让 Codex 按参考图风格或对象进行处理
- Codex 任务交接：Pedit 会生成清晰的 handoff 指令，帮助 Codex 接手任务
- 导出图片：可以导出当前版本图片

## 安装要求

你需要先准备：

- Codex desktop，并启用插件能力
- Node.js 20 或更新版本
- pnpm

## 从 GitHub 安装

克隆项目：

```bash
git clone https://github.com/812745450-png/pedit.git
cd pedit
```

安装依赖并构建：

```bash
pnpm install
pnpm build
pnpm validate:plugin
```

然后在 Codex 中把这个文件夹作为本地插件安装或启用。启用后，建议新开一个 Codex 线程，再输入：

```text
@pedit 打开
```

如果插件没有立刻出现，通常是因为 Codex 线程还没有重新加载 MCP 工具。重新打开 Codex 线程后再试一次。

## 使用流程

1. 在 Codex 中输入 `@pedit 打开`
2. 在 Pedit 里上传图片，创建项目
3. 进入图片详情页，点击进入编辑
4. 根据任务选择操作方式：
   - 不圈选区域：直接输入整图修图指令
   - 圈选区域：用标注工具圈出需要修改的位置，再输入局部指令
   - 有参考图：上传参考图，并在指令中说明要参考什么
5. 点击开始优化
6. 复制 Pedit 生成的 Codex handoff 指令，粘贴到 Codex 对话并发送
7. Codex 完成修图后，结果会写回 Pedit，成为新的版本节点
8. 在版本树里对比、继续编辑或导出最终图片

## 两种修图模式

### 整图编辑

适合没有明确选区的任务，比如：

- 整体增强画质
- 调整照片氛围
- 换整体背景
- 让画面更自然、更高级
- 按参考图统一风格

这类任务不需要手动画 mask，也不要把局部圈选当成限制。

### 局部编辑

适合需要严格控制范围的任务，比如：

- 只修改包装上的文字
- 只改变衣服颜色
- 只去除某个小物体
- 只调整某个区域的细节

如果是局部换字、去字、换包装这类任务，建议先让 Codex 消除原来的内容，保证背景衔接自然，再添加新元素或新文字。

## 使用建议

- 指令尽量写清楚“保留什么”和“修改什么”
- 局部修图时，选区尽量覆盖完整目标，不要只圈一小部分
- 换文字时，要说明原文字需要先清除干净
- 有参考图时，要说明参考的是风格、构图、颜色、产品形态还是具体对象
- 生成结果不满意时，可以基于当前版本继续修，而不是回到原图重来

## 项目和隐私

Pedit 的项目、上传图片、生成图片和任务状态默认保存在本机。请不要把这些目录上传到公开仓库：

```text
.pedit-runtime/
packages/server/.pedit-runtime/
docs/validation/
```

当前 GitHub 仓库已经通过 `.gitignore` 排除了这些本地数据。

## 当前状态

Pedit 目前还是 alpha 版本，适合小范围试用和继续迭代。核心流程已经可以跑通，但安装、授权、插件启用、Codex MCP 工具加载等步骤还可能受本机环境影响。

如果你只是想试用，建议先从 GitHub Release 下载 `pedit-release.zip`，或直接 clone 仓库后按上面的步骤构建。
